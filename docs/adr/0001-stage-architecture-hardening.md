# Stage architecture hardening

NoNameTag will address the architecture and performance review in two implementation stages: first fixing correctness, configuration drift, and player stats cache release; then extracting chat message handling behind a testable sender seam and unifying death attribution behind a resolver and shared context. This staged approach keeps each change reviewable while still paying down the architectural debt that caused the same runtime concepts to be spread across the plugin entrypoint and service layer. Each stage must be implemented with TDD and reviewed by separate Spec and Standards subagents before it is considered complete. The chat extraction should include a dedicated sender seam so chat routing behavior can be tested without a live Rocket/Unturned runtime. Death attribution should likewise be resolved once through a dedicated resolver so player stats and death messages consume the same attribution result.
